Frequently Asked Questions

What is a hash function?

A hash function takes input data of any size and returns a fixed-size string of characters, which typically appears random. The same input will always produce the same hash, but even a small change in input will produce a completely different hash.

Which hash algorithm should I use?

It depends on your use case:
- MD5: Fast but cryptographically broken, good for checksums
- SHA1: More secure than MD5 but also considered weak
- SHA256: Currently recommended for most security purposes
- SHA512: More secure but generates longer hashes
For password hashing, consider specialized algorithms like bcrypt instead.

Can I use this for password hashing?

While you can generate hashes of passwords, these algorithms alone are not recommended for password storage. For passwords, use specialized password hashing algorithms like bcrypt, scrypt, or PBKDF2 which include salting and are computationally intensive.

How are file hashes generated?

The file is read as binary data and processed through the hash algorithm. The resulting hash serves as a unique fingerprint for that exact file content.

Is there a file size limit for hashing?

There's no strict limit, but very large files may cause performance issues in your browser. For best results, we recommend files under 100MB.
